DATABASE


DATA

Data are raw facts and figures that have no meaning on their own. Example: "25", "Red", "Kathmandu", "Monday"

INFORMATION

Information is processed or organized data that is meaningful and useful for decision-making. Example: "The temperature in Kathmandu is 25°C on Monday."

Features of Information:

Good information should have the following features:
  • Accuracy – Correct and reliable.
  • Relevance – Related to the purpose.
  • Timeliness – Available when needed.
  • Completeness – Includes all necessary details.
  • Clarity – Easy to understand.
  • Cost-effectiveness – Benefits should outweigh the cost of getting it.

DATABASE

A database is an organized collection of related data stored in a structured way, usually in electronic form, that can be easily accessed, managed, and updated.

PURPOSE OF DATA BASE

  • To store data in an organized way.
  • To reduce data duplication and inconsistency.
  • To allow easy access, retrieval, and manipulation of data.
  • To secure data and maintain integrity.
  • To support decision-making with timely information.

Terminologies in Database

  • Table: A collection of related data organized in rows and columns.
  • Field (Attribute): A column in a table representing a data type.
  • Record (Row/Tuple): A single entry in a table, consisting of fields.
  • Tuple: Synonym for a record; a single data item in a relation.
  • Object: An instance of a class in object-oriented databases; data plus behavior.
  • Keys: Unique identifiers used to access records (Primary Key, Foreign Key, Candidate Key).

Data Dictionary

A data dictionary is a centralized repository of information about data such as meaning, relationships to other data, origin, usage, and format. It helps users and administrators understand database structure, metadata, and constraints.

Database Management System (DBMS)

Introduction:

A Database Management System (DBMS) is software that allows users to define, manipulate, retrieve, and manage data efficiently. It acts as an interface between users and the database.

Objectives:

  • Organize and store large volumes of data.
  • Allow multiple users to access data concurrently.
  • Ensure data integrity and security.
  • Minimize data redundancy.
  • Enable easy data retrieval and reporting.

Advantages:

  • Reduced data redundancy
  • Improved data integrity and consistency
  • Easy data access and management
  • Multi-user access control
  • Data security and backup features

Disadvantages:

  • Complex and costly to implement
  • Requires high technical knowledge
  • Increased risk if system crashes
  • Regular maintenance required

Types of Database Models

  • Hierarchical Model:
    • Data is organized in a tree-like structure.
    • Each child has only one parent.
    • Example: File directory systems.
  • Network Model:
    • Data is represented as records and relationships as links.
    • A record can have multiple parent and child records.
    • Example: Telecom networks.
  • Relational Model:
    • Data is stored in tables (relations) with rows and columns.
    • Uses SQL for querying.
    • Most widely used.
  • Entity-Relationship Model (ER Model):
    • Represents real-world entities and their relationships.
    • Used during database design to visualize schema structure.

Integrity Constraints and Their Types

  • Domain Constraint: Specifies the allowed values for a given attribute.
  • Entity Integrity: Each table must have a primary key; values cannot be null.
  • Referential Integrity: Foreign key values must match primary key values in the referenced table or be null.

Keys

  • Primary Key: Uniquely identifies each record.
  • Foreign Key: Links one table to another.
  • Candidate Key: Potential primary keys.
  • Composite Key: Multiple fields together uniquely identify a row.

Normalization

Normalization is the process of organizing data to reduce redundancy and improve data integrity.

  • 1NF (First Normal Form): No repeating groups or arrays.
  • 2NF (Second Normal Form): 1NF + No partial dependency.
  • 3NF (Third Normal Form): 2NF + No transitive dependency.

Advantages:

  • Removes data redundancy
  • Ensures data integrity
  • Simplifies data maintenance

Disadvantages:

  • Complex joins for queries
  • More tables to manage

Centralized Database

Introduction:

All data is stored at a single location.

Advantages:

  • Easier to maintain
  • Simple backup and recovery

Disadvantages:

  • Single point of failure
  • Slow access for remote users

Distributed Database

Introduction:

Data is distributed across multiple physical locations.

Advantages:

  • Better reliability and availability
  • Faster local access

Disadvantages:

  • Complex management
  • Expensive setup

Challenges:

Security Measures:

DDL (Data Definition Language)

Used to define and modify the structure of database objects.

Examples: CREATE, ALTER, DROP

DML (Data Manipulation Language)

Used to manipulate data within tables.

Examples: SELECT, INSERT, UPDATE, DELETE

Database Security

Introduction

Database security refers to the collection of tools, controls, and measures designed to establish and preserve the confidentiality, integrity, and availability of the data stored in a database. Since databases store critical organizational information (such as financial records, personal details, research data, etc.), ensuring their protection from unauthorized access, misuse, or corruption is essential.


Key aspects:
  • Confidentiality: Protecting data from unauthorized access.
  • Integrity: Ensuring data remains accurate and consistent.
  • Availability: Making sure data is accessible to authorized users when needed.

Challenges in database security

  1. Unauthorized Access – Hackers or insiders gaining access to sensitive information.
  2. SQL Injection Attacks – Malicious code inserted into queries to manipulate databases.
  3. Insider Threats – Employees or users misusing their privileges.
  4. Data Leakage – Sensitive data being exposed through poor controls.
  5. Weak Authentication – Use of simple or shared passwords making systems vulnerable.
  6. Malware and Ransomware Attacks – Compromising database files or systems.
  7. Backup Security – Improperly secured backups being stolen or tampered with.
  8. Compliance Requirements – Meeting legal and regulatory standards like GDPR, HIPAA, etc.

Security Measures

To protect databases, organizations adopt the following measures:

  1. Access Control
    • Implement strong authentication (passwords, biometrics, MFA).
    • Role-based access (least privilege principle).
  2. Encryption
    • Encrypt data at rest and in transit.
    • Use SSL/TLS for communication.
  3. Database Activity Monitoring
    • Real-time monitoring of queries and access patterns.
    • Alert on suspicious activities.
  4. Regular Updates & Patching
    • Keep database software updated to fix vulnerabilities.
  5. Backup & Recovery Plans
    • Securely store backups.
    • Test recovery mechanisms regularly.

ER Diagrams

Introduction

  • An Entity–Relationship (ER) Diagram is a visual representation of data and the relationships between entities in a database.
  • It was introduced by Peter Chen (1976) and is widely used in database design to model the logical structure before implementation.
  • Helps in understanding how data is stored, organized, and related in a database system.

Entity

  • An Entity represents a real-world object, concept, or thing about which data is stored.
  • Entities can be:
    • Strong Entity – has a primary key and exists independently (e.g., Student, Employee).
    • Weak Entity – depends on another entity for existence, identified by a partial key (e.g., Dependent, OrderItem).
  • Symbol: Rectangle (Strong Entity), Double Rectangle (Weak Entity).

Attributes

  • Attributes are properties or characteristics of an entity or relationship.
  • Types of attributes:
    • Simple Attribute – indivisible (e.g., Name, Age).
    • Composite Attribute – can be divided into smaller parts (e.g., FullName → FirstName, LastName).
    • Derived Attribute – value can be derived from another attribute (e.g., Age from DateOfBirth).
    • Multi-valued Attribute – can hold multiple values (e.g., Phone Numbers).
  • Symbol: Ellipse (Oval). Double Oval for multi-valued, Dashed Oval for derived.

Relationships

  • A Relationship shows how two or more entities are connected.
  • Types of Relationships:
    • One-to-One (1:1) – One entity is associated with only one entity of another set.
    • One-to-Many (1:N) – One entity is related to many entities of another set.
    • Many-to-Many (M:N) – Many entities are related to many entities of another set.
  • Symbol: Diamond

Cardinality and Degree

  • • Cardinality – Defines the number of instances of one entity that can be associated with another entity.
    • 1:1 → A student has one ID card.
    • 1:N → A teacher teaches many students.
    • M:N → Students enroll in many courses, and each course has many students.

Degree of Relationship

  • Degree of Relationship – Number of entities participating in a relationship.
  • Types of Degree:
    • Unary (1 entity) – Employee supervises Employee.
    • Binary (2 entities) – Student enrolled in Course.
    • Ternary (3 entities) – Doctor prescribes Medicine to Patient.

SQL Concepts

1. Data Constraints

  • NOT NULL – Ensures a column cannot have NULL values.
  • UNIQUE – Ensures all values in a column are distinct.
  • PRIMARY KEY – Uniquely identifies each row in a table (combines NOT NULL + UNIQUE).
  • AUTO INCREMENT – Automatically generates a unique numeric value for each new row (commonly used for primary keys).

2. Operators

  • AND – Returns TRUE if both conditions are TRUE.
  • OR – Returns TRUE if at least one condition is TRUE.
  • NOT – Reverses the result of a condition.

3. WHERE Clause

Used to filter records based on a specified condition.

Example:

SELECT * FROM Employees
WHERE Salary > 50000;

4. ORDER BY

Sorts query results in ascending (ASC) or descending (DESC) order.

Example:

SELECT * FROM Students
ORDER BY Name ASC;
