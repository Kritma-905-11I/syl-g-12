PAST PAPER Q/A OF DBMS

1. What is Database and DBMS? List out the advantages and disadvantages of DBMS.

A database is an organized collection of data that is stored and accessed electronically. It allows data to be efficiently retrieved, managed, and updated. Databases are used in various applications, from websites and mobile apps to banking systems and school management software.

A Database Management System (DBMS) is software that interacts with users, applications, and the database itself to capture and analyze data. It helps users create, manage, and manipulate databases without needing to know how the data is physically stored. Examples of DBMS: MySQL, Oracle, Microsoft SQL Server, PostgreSQL, SQLite, etc.

Advantages of DBMS:

  • Data Redundancy Control – Prevents duplication of data by storing data in one central location.
  • Data Integrity and Accuracy – Ensures consistent and accurate data by enforcing rules and constraints.
  • Data Security – Provides user-based access control to protect sensitive data.
  • Backup and Recovery – Offers automatic backup and recovery options in case of system failure.
  • Data Sharing – Multiple users can access the database simultaneously without conflict.
  • Data Independence – Changes in data structure do not affect the application programs.
  • Efficient Query Processing – Allows quick retrieval and processing of data using SQL or other query languages.

Disadvantages of DBMS:

  • High Cost – Installation, setup, and maintenance of DBMS software and hardware can be expensive.
  • Complexity – Requires skilled professionals to design, manage, and maintain databases.
  • Performance Overhead – Large DBMS systems can slow down due to processing overhead.
  • Size and Storage – DBMS systems require significant memory and storage.
  • Risk of Data Loss – Although backups are available, improper handling or attacks can still cause data loss or corruption.

2. Differentiate between file processing system and DBMS. Give at least four points.

Difference between File Processing System and DBMS:

  • Storage:
    • File Processing System: Data is stored in separate files for each application.
    • DBMS: Data is stored in a centralized database accessible by multiple applications.
  • Redundancy:
    • File Processing System: High data redundancy and inconsistency.
    • DBMS: Reduces redundancy and maintains consistency.
  • Security:
    • File Processing System: Difficult to implement security and access control.
    • DBMS: Provides strong security features like access control and user permissions.
  • Query Support:
    • File Processing System: No support for complex queries; data access is manual or sequential.
    • DBMS: Supports complex queries using SQL and advanced data operations.

Explain the different models of DBMS with advantages and disadvantages.

Different Models of DBMS

1. Hierarchical Model

This model organizes data in a tree-like structure with a single root and multiple levels of related records.

Advantages:
  • Easy to understand and use.
  • Efficient for queries with a predictable hierarchy.
  • Data integrity is maintained due to parent-child relationships.
Disadvantages:
  • Complex to reorganize or modify structure.
  • Redundancy may occur if relationships are many-to-many.
  • Not flexible for dynamic or changing relationships.

2. Network Model

This model represents data as records connected through links (pointers), allowing many-to-many relationships.

Advantages:
  • More flexible than the hierarchical model.
  • Efficient for complex relationships among data.
  • Faster data access using pointers.
Disadvantages:
  • Complex structure and navigation.
  • Requires detailed knowledge of the links for querying.
  • Harder to maintain and modify.

3. Relational Model

Data is organized in tables (relations) with rows and columns. Most widely used model today.

Advantages:
  • Simple and easy to understand.
  • Flexible and powerful querying using SQL.
  • Reduces data redundancy and ensures consistency.
Disadvantages:
  • Performance can be slower for very large databases.
  • Requires complex joins for multi-table queries.
  • Can consume more memory for relational operations.

4. Object-Oriented Model

Data is represented as objects, similar to object-oriented programming languages.

Advantages:
  • Supports complex data types like images, videos, and user-defined types.
  • Encapsulation and inheritance increase reusability and organization.
  • Better mapping between application and database objects.
Disadvantages:
  • More complex to design and understand.
  • Not as widely supported as relational model.
  • Can be slower for simple data operations.

What is a relational database? How is it different from other database models?

What is a Relational Database?

A relational database is a type of database that stores data in tables (also called relations), made up of rows and columns. Each row is a record, and each column is a field or attribute. Tables can be linked using foreign keys to establish relationships between data.

Relational databases use SQL (Structured Query Language) for storing, updating, and querying data.

Examples: MySQL, PostgreSQL, Oracle, SQL Server, SQLite.

Summary

  • Relational databases are simple, scalable, and widely used.
  • They use SQL for easy data management.
  • Support strong data integrity and relationships between data.
